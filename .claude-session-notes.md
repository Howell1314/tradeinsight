## 协作约束（必读）

- Zustand persist 清理副作用必须走 `useAuthStore.initialized === true` 守卫
- 云同步写入必须用 `trackSync(x).catch(catchSync('label'))` 模式（见 `src/lib/pendingSync.ts`）；`syncTrades.ts` / `syncPlans.ts` 里每个 Supabase 调用必须 `const { error } = await ...; if (error) throw error`——裸 `await supabase.from(...)` 会被 `npm run lint:supabase-errors` 拦住（已接入 build）
- ID 使用确定性规则（沿用 `generateHotSectorId` / `generateHotStockId` / `generatePlanId` 模式）
- 可空单行查询用 `maybeSingle()` 不用 `single()`
- 优先扩展现有 JSONB，非必要不加新列不做 migration
- 一次只改一个文件，diff 摘要给用户 review 通过再进下一个
- typecheck 必须用 `npm run typecheck`（内含 `tsc -b --noEmit`），不要用 `tsc --noEmit`
- 每个 Block 完成后分 commit push，不攒大 PR
