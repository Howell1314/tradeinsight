## 1. 项目概述

**TradeInsight** 是一款面向个人投资者的全能交易记录与分析工具，支持多资产类别、多账户管理，并提供专业级的绩效统计与交易日志功能。

### 核心目标

- 让交易者快速录入、管理所有交易记录，无需依赖 Excel
- 通过 FIFO 自动撮合算法，精确计算每笔盈亏（含手续费）
- 提供多维度统计分析，帮助交易者发现自身盈利模式和失误规律
- 通过交易日志与复盘机制，推动持续改进
- 数据本地优先（LocalStorage），登录后自动云端同步（Supabase）

### 支持的资产类别

| 标识 | 名称 |
|------|------|
| `crypto` | 数字货币 |
| `equity` | 美股个股 |
| `option` | 期权 |
| `etf` | ETF |
| `cfd` | CFD 差价合约 |
| `futures` | 期货 |

### 数据流向

1. **未登录**：所有数据存储在浏览器 LocalStorage，完全本地离线可用。
2. **登录时**：并行拉取 trades、accounts、account_transactions、risk_rules、journal_entries，与本地数据进行 last-write-wins 合并：
   - 有 `updated_at` 字段的数据：比较时间戳，取较新的一方
   - 本地独有（云端没有）的记录：自动推送至云端
   - 云端独有（本地没有）的记录：拉取到本地
3. **运行时**：每次增删改操作同时写入 LocalStorage（通过 zustand persist）并异步同步至 Supabase。
4. **退出登录**：清空内存中的用户数据（trades、journal、prices 等），LocalStorage 恢复为空。

## 4. 数据模型

### 4.1 交易记录（Trade）

```typescript
interface Trade {
  id: string              // 唯一 ID（时间戳 + 随机串）
  account_id: string      // 所属账户 ID
  asset_class: AssetClass // 资产类别
  symbol: string          // 标的代码（如 AAPL、BTC）
  direction: Direction    // buy | sell | short | cover
  quantity: number        // 成交数量（>0）
  price: number           // 成交价格
  total_amount: number    // 成交总额
  commission: number      // 佣金/手续费
  fees: number            // 其他费用（如交易所规费）
  currency: string        // 货币单位（如 USD）
  traded_at: string       // 成交时间（ISO 8601）
  strategy_tags: string[] // 策略标签
  notes: string           // 备注（≤5000 字）
  emotion?: EmotionTag    // 情绪标签
  planned_stop?: number   // 计划止损价（用于计算 R 倍数）
  planned_target?: number // 计划目标价
  metadata: Record<string, unknown> // 扩展元数据（含 contract_multiplier、expiration 等）
  created_at: string      // 创建时间
  updated_at?: string     // 最后修改时间（云端同步冲突解决用）
}
```

**交易方向语义：**

| direction | 含义 | 效果 |
|-----------|------|------|
| `buy` | 买入做多 | 开或加多头仓位 |
| `sell` | 卖出平多 | 减或平多头仓位，触发 FIFO 撮合 |
| `short` | 做空 | 开或加空头仓位 |
| `cover` | 回补（平空）| 减或平空头仓位，触发 FIFO 撮合 |

```typescript
interface Account {
```

由 FIFO 引擎自动计算，**非用户直接录入**：

```typescript
interface ClosedTrade {

  symbol: string
  asset_class: AssetClass
  account_id: string
  direction: 'long' | 'short'
  open_trades: Trade[]    // 构成开仓的原始交易
  close_trades: Trade[]   // 构成平仓的原始交易
  open_price: number
  close_price: number
  quantity: number
  gross_pnl: number       // 毛利润
  net_pnl: number         // 净利润（扣除手续费和其他费用）
  commission: number
  fees: number
  opened_at: string
  closed_at: string
  holding_days: number    // 持仓天数
```

## 5. 功能模块详解

### 5.1 仪表盘（Dashboard）

核心概览页，展示所有关键指标和图表。

**统计指标（第一行）：**

| 指标 | 说明 |
|------|------|
| 总盈亏 | 已实现 + 未实现盈亏 |
| 胜率 | 盈利笔数 / 总结算笔数 |
| 盈亏比 | 平均盈利 / 平均亏损 |
| 最大回撤 | 权益峰值到谷值的最大跌幅 |

**统计指标（第二行）：**

| 指标 | 说明 |
|------|------|
| 期望值 | 胜率 × 平均盈利 − 败率 × 平均亏损 |
| 利润因子 | 总利润 / 总亏损 |
| 夏普比率 | 先按交易日聚合 PnL，再计算均值/标准差 × √252 年化 |
| 平均持仓 | 所有已结算交易的平均持仓天数 |
| 总回报率 | 净盈亏 / 总投入资金 |

**图表：**
- 累计收益曲线（面积图，含滑动缩放滑块）
- 品种盈亏分布（甜甜圈饼图）
- 按星期分析（柱状图，正负值用绿/红区分）
- 交易统计快览（连胜/连亏/持仓数等）

**风险规则预警：**
- 触发单日最大亏损限制时显示红色警告条
- 达成月度盈利目标时显示绿色提示条

### 5.2 交易记录（Trades）

原始交易流水管理页，所有操作均会触发重新计算持仓和统计。

**功能列表：**
- **添加交易**：通过弹窗录入，支持所有资产类别和方向，支持计划止损/目标价录入
- **编辑交易**：直接修改已录入记录
- **删除交易**：删除后重新撮合计算
- **卖出/回补校验**：若卖出数量超过当前多头持仓，系统阻止并提示错误
- **批量 CSV 导入**：上传标准格式 CSV，自动映射字段
- **导出 CSV**：将当前筛选结果导出为格式化的 CSV 文件

**导出 CSV 包含字段（v2.0 优化后）：**

| 列名 | 说明 |
|------|------|
| 日期 | `YYYY-MM-DD` 格式 |
| 品种 | 资产类别中文名 |
| 标的 | symbol |
| 方向 | 中文（买入/卖出/做空/回补）|
| 数量 | 整数不带小数，小数保留4位 |
| 价格 | 保留2位小数 |
| 金额 | 保留2位小数 |
| 手续费 | 保留2位小数 |
| 其他费用 | fees，保留2位小数 |
| 账户 | 账户名称 |
| 合约乘数 | — |
| 到期日 | `YYYY-MM-DD` 格式（正确解析原始 Date 字符串）|
| 计划止损 | 有则填写，保留2位小数 |
| 计划目标 | 有则填写，保留2位小数 |
| 情绪 | 中文情绪标签 |
| 策略标签 | 多个用 ` / ` 分隔 |
| 备注 | — |

**筛选条件：**
- 关键词搜索（标的代码 + 备注）
- 资产类别筛选
- 方向筛选
- 月份筛选
- 自定义日期范围

**分页：** 每页 20 条，支持翻页。

### 5.3 持仓管理（Positions）

显示当前所有未平仓持仓，数据由 FIFO 引擎自动维护。

**功能：**
- 多头/空头持仓分开展示
- **一键刷新行情**：并行请求所有持仓的当前价格，三级 fallback 保障可用性：
  1. Yahoo Finance `query1`（直连）
  2. Yahoo Finance `query2`（备用域名）
  3. `corsproxy.io`（CORS 代理兜底）
- 可手动输入当前市价，实时计算浮动盈亏
- 合约乘数自动纳入盈亏计算（期货/期权）
- **价格过期提示（v2.0 新增）**：
  - 距上次更新 < 30 分钟：不显示（新鲜）
  - 30 分钟 ~ 4 小时：灰色显示"X分钟前 / X小时前"
  - 超过 4 小时：橙色 ⚠ 警告提示

### 5.4 统计分析（Analytics）

五标签深度分析页。

#### 综合分析（overview）
- 完整累计收益曲线（支持点击查看当日交易明细）
- 8 项关键指标卡片
- 各品种盈亏横向对比条

#### 标的分析（symbols）
- Top 15 标的盈亏排行（水平柱状图）
- 各标的详细统计表（交易次数、胜率、总盈亏、均盈亏）

#### 时间分析（time）
- 月度盈亏柱状图
- 星期盈亏分布
- 月度胜率折线趋势图
- 季度盈亏对比
- 月度统计汇总表

#### 策略分析（strategy）
- 各策略标签的胜率、盈亏比、期望值
- 策略月度 PnL 趋势折线图

#### 风险指标（risk）
- 回撤曲线
- R 倍数分布（需录入计划止损价）
- 情绪状态与盈亏关联分析
- **失误标签成本分析**（v2.0 改进）：从 `useJournalStore` 响应式读取，不再直读 localStorage

### 5.5 交易日志（Journal）

结构化交易日记，帮助交易者养成复盘习惯。

**日志字段：**
- 日期、当日盈亏（手动填写）
- 情绪状态（冷静/自信/犹豫/冲动，彩色标签）
- 当日总结、做得好的、需要改进的、明日计划
- 失误标签（多选：追涨杀跌/过早止盈/未止损/仓位过重/情绪交易/违反计划/错误判断方向/持仓太久）
- 截图上传（最多 5 张，单张 ≤10MB）
- 复盘笔记（独立保存区域）

**联动特性：** 查看日志详情时，自动关联并展示该日期的已结算交易记录。

**搜索与筛选：** 支持按关键词、情绪、失误标签、日期范围筛选。

**云端同步（v2.0 重构）：**
- Journal 数据迁移至独立的 `useJournalStore`（Zustand + persist）
- 登录时在 `App.tsx` 统一触发 `syncJournal`，与 trade 数据并行拉取
- 合并策略：last-write-wins（比较 `updated_at`，本地较新时回推云端）

## 7. 核心计算逻辑

### 7.1 FIFO 撮合算法（`processTradesIntoPositions`）

所有交易按时间排序后，按标的 + 账户分组，分别维护多头队列（`longQueue`）和空头队列（`shortQueue`）：

```
买入（buy）：
  1. 优先撮合空头队列（平空）→ 生成 ClosedTrade（short）
  2. 剩余数量加入多头队列

卖出（sell）：
  1. 优先撮合多头队列（平多）→ 生成 ClosedTrade（long）
  2. 剩余数量加入空头队列（开新空仓）

做空（short）→ 直接加入空头队列
回补（cover）→ 优先撮合空头队列，剩余加入多头队列
```

盈亏计算：
- 多头：`gross_pnl = (close_price - open_price) × qty × multiplier`
- 空头：`gross_pnl = (open_price - close_price) × qty × multiplier`
- `net_pnl = gross_pnl - 按比例分摊的开平仓（commission + fees）`

分批建仓均价：`avg_cost = Σ(price × remaining) / Σremaining`（数量加权）

### 7.2 derive() 缓存机制（v2.0 新增）

```typescript
// 模块级缓存
let _coreCache: { trades; selectedAccount; closedTrades; rawPositions; stats } | null = null

function deriveCore(trades, selectedAccount) {
  // trades 引用相同 且 selectedAccount 相同 → 直接返回缓存
  if (_coreCache?.trades === trades && _coreCache.selectedAccount === selectedAccount)
    return _coreCache
  // 否则全量重算并缓存
}

function derive(trades, selectedAccount, currentPrices) {
  const { closedTrades, rawPositions, stats } = deriveCore(trades, selectedAccount)
  // 只对 rawPositions 应用 currentPrices（轻量操作）
  return { closedTrades, openPositions: applyCurrentPrices(rawPositions, currentPrices), stats }
}
```

价格刷新（`updateCurrentPrice`）不经过 `derive`，直接调用 `applyCurrentPrices`，完全绕过重算。

### 7.3 统计指标计算（`computeStats`）

| 指标 | 计算公式 |
|------|---------|
| 胜率 | `盈利笔数 / 总笔数` |
| 盈亏比 | `平均盈利 / 平均亏损` |
| 期望值 | `胜率 × 平均盈利 - 败率 × 平均亏损` |
| 最大回撤 | 逐笔累加净值，记录峰值，`max((峰值 - 当前净值) / 峰值)` |
| 利润因子 | `总盈利 / 总亏损` |
| 夏普比率 | 先按交易日聚合 PnL → 计算日均值/日标准差 → × √252 年化 |
| 连胜/连亏 | 遍历时序记录连续次数峰值 |

### 7.4 浮动盈亏计算

```
多头：unrealized_pnl = (current_price - avg_cost) × |qty| × multiplier
空头：unrealized_pnl = (avg_cost - current_price) × |qty| × multiplier
unrealized_pnl_pct = unrealized_pnl / (avg_cost × |qty|) × 100%
```

### 7.5 R 倍数计算

```
initial_risk（多头）= (open_price - planned_stop) × qty × multiplier
initial_risk（空头）= (planned_stop - open_price) × qty × multiplier
actual_r = net_pnl / initial_risk
```

## 8. 安全机制

### 输入验证

所有从 LocalStorage 恢复的数据在使用前通过 `isValidTrade` / `isValidAccount` 校验：

- 字段类型校验（string / number / array）
- 数值范围校验（price ≥ 0，quantity > 0，isFinite 检测）
- 字符串长度上限（symbol ≤ 30，notes ≤ 5001，id ≤ 64）
- 枚举白名单校验（asset_class、direction 必须在允许集合内）
- metadata 必须为非 null 的普通对象（防止原型污染）

日志条目通过 `isValidEntry` 校验：
- 日期格式：`/^\d{4}-\d{2}-\d{2}$/`
- 情绪标签仅允许：`['冷静', '自信', '犹豫', '冲动']`
- 图片 URL 数量 ≤ 10，单个 URL 长度 ≤ 600

### 业务规则校验

- `validateNewTrade`：卖出/回补前检查当前持仓是否充足，防止超卖/超平

### 文件上传限制

- 日志截图单张 ≤ 10MB，仅接受 `image/*` MIME 类型
- 每条日志最多 5 张图片

### Supabase 安全

- 所有数据库操作均通过 Row Level Security（RLS）绑定 `user_id`
- 客户端使用 `@supabase/supabase-js` 官方 SDK，不直接拼接 SQL
